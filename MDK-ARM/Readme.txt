更新202600512;
1、问题：脉冲定时器TIM2，首脉冲异常过宽！！！    
   原因：在OC影子寄存器值更新之前、输入捕获库函数“HAL_TIM_OC_Start_IT(&htim2,TIM_CHANNEL_2);”提前开启了定时器计数，导致比较值更新滞后。
         同时，首脉冲宽度为固定值，没有动态调整，每次启用OC功能就需要重新配置通道脉冲初始值。
   解决方式:（1）、更改向下计数方式
           （2）、将OC影子寄存器值更新放在定时器启动之前，输入捕获只使能中断，不开启计数器。
           （3）、为方便使用IC功能动态调整脉宽，设定在下一个更新事件后改变（output compare preload ==ENABLE）,、
                  这导致首脉冲OC比较值为固定初始值，因此首脉冲采用立即更新的方式。
                  代码如下：
                  {
                  __HAL_TIM_DISABLE_OCxPRELOAD(&htim2, TIM_CHANNEL_2);//先立即更新
	               __HAL_TIM_SetCompare(&htim2,TIM_CHANNEL_2,timeLoad100ns-1); //MAX pulse 500us
	               __HAL_TIM_ENABLE_OCxPRELOAD(&htim2, TIM_CHANNEL_2);//wait next update event
                  }
            提醒：如需脉宽调整在当前脉冲，可采用方案如下：
                  (A)、计数方式改为向上计数，
                  (B)、脉冲OC值立即更新“ __HAL_TIM_DISABLE_OCxPRELOAD(&htim2, TIM_CHANNEL_2);”
                  (C)、如此可实现在当前脉冲动态调整，本软件版本采用下一个周期更新即可。！！
